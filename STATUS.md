## 💰 Current cost

Authoritative source: [`docs/COST_MODEL_AND_SCALING.md`](./docs/COST_MODEL_AND_SCALING.md) (Apr 18, 2026 baseline) and [`docs/cost/consumption-vs-reserved-analysis.md`](./docs/cost/consumption-vs-reserved-analysis.md) (Apr 30, 2026 — B1 vs Container Apps modeling).

| Environment | Major line items | ~Monthly |
|---|---|---|
| **Production** | App Service B1 ($13.14) + SQL Basic 5DTU ($4.90) + KV/AppInsights/Logs/alerts (~$2.50) + GHCR/network (~$0.50) | **~$21** |
| **Staging** | App Service B1 ($13.14) + SQL Free (≈$0) + KV/AppInsights/Logs/storage (~$10) | **~$23** |
| GHCR / GitHub Actions | within free org allotment for current usage | $0 |
| **Total Azure** | | **~$44–53 / mo** |

**B1-vs-Container-Apps consumption decision (bd `j6tq`, 2026-04-30):** B1 is the right SKU for now. 17+ background schedulers (4 of them hourly) keep the app continuously warm — naive Container Apps consumption migration would cost *more*, not less. A split architecture (API-as-Container-App + schedulers-as-Container-App-Jobs) saves ~$5/mo at breakeven, but migration cost (20–40 eng-hrs) means payback measured in years. Decision-ready, no migration recommended.

**ROI history:** infrastructure was $298/mo before April 2026 cost-optimization sweep. 75% reduction landed via B2→B1, S2→Basic, ACR deletion, and orphan cleanup.
