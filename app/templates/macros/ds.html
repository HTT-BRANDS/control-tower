Why `{% import ... as _ns %}` + `{% set alias = _ns.macro %}`? Jinja's `{% from x import y %}` binds `y` into the importing template's RENDER-TIME locals but does NOT publish `y` on the template's exported module namespace — so a caller `{% from "macros/ds.html" import ds_card %}` would see nothing. `{% include %}` has the same blind spot. The `{% set alias = _ns.macro %}` pattern publishes each macro at module level, which Jinja DOES export, and transparently forwards `{% call %}` callers because Jinja macros are first-class values.
